# 职配助手项目进度文档

更新时间：2026-07-07

## 1. 项目定位

职配助手是一款面向求职者与招聘方的桌面端职业匹配分析工具。当前版本基于 Vite + React + TypeScript + Tauri 开发，核心能力包括求职诊断、招聘需求分析、简历优化、AI 辅助分析、历史记录、本地隐私管理和桌面端发布。

## 2. 当前进度结论

当前项目处于“可内测交付 + 持续产品化优化”阶段。

也就是说，核心功能已经可用，桌面版发布链路已经打通，正式 exe 不再弹出 CMD 黑框，基础自动化测试、构建检查、安全审计和桌面发布检查都已通过。现在不再是早期原型阶段，而是进入体验打磨、组件拆分、测试扩展和交付包装阶段。

## 3. 已完成内容

### 3.1 桌面端迁移与发布

- 已将压缩包项目解压并整理到当前项目目录。
- 已完成 Tauri 桌面端基础运行与构建配置。
- 已修复 Windows 正式版启动时弹出 CMD 控制台窗口的问题。
- 已新增桌面发布检查脚本，覆盖 Windows GUI 子系统、NSIS 安装器、图标、dist 资源等关键项。
- 已维护版本一致性检查，确保 `package.json`、Tauri、Rust、更新公告版本一致。

### 3.2 核心产品功能

- 已完成求职者工作流：角色选择、模式选择、问答填写、本地分析、AI 分析、结果保存。
- 已完成招聘方工作流：岗位需求分析、JD 优化、候选人简历筛选、面试追问建议。
- 已完成学生模式与无实习方向推荐。
- 已完成简历文本导入、简历预览、简历改写建议和多版本简历策略。
- 已完成历史记录、个人档案、隐私与数据管理页面。

### 3.3 AI 配置与隐私

- 已新增 AI 配置教程，包含服务商预设、API Key、Base URL、模型、连接测试和隐私授权说明。
- 已增加 AI 连接测试前置校验，未填写 Key 时不会误触发测试。
- 已增加 AI Key 清除入口。
- 已增强错误信息处理，避免直接暴露过长或敏感的底层错误。
- 已确保导出本地数据时不会导出 AI API Key。

### 3.4 示例数据体验

- 已强化“这是示例内容”的提示。
- 已新增“一键清空示例”能力。
- 已修复一键清空按钮行为，确保不会误提交表单。
- 已增加示例数据识别与清空逻辑测试。

### 3.5 题目与分析质量

- 已从心理学家、职业规划师、HR、求职者四个视角优化题目。
- 心理测评题已改为行为锚定式问题，减少抽象自评和价值判断。
- 职业规划问题已强化目标清晰度、现实边界、市场验证、证据缺口。
- HR 侧问题已强化硬性要求、可培养项、淘汰项、面试追问证据。
- 已加入招聘合规提醒，避免使用年龄、性别、婚育、籍贯等身份标签作为筛选标准。

### 3.6 代码结构优化

`App.tsx` 已从约 3700 多行逐步瘦身到约 2613 行。

已拆出的模块包括：

- `src/questions.tsx`：问答模块配置。
- `src/sampleData.ts`：示例数据识别与清空。
- `src/errors.ts`：错误信息清洗。
- `src/aiResult.tsx`：AI 结构化结果、质量检查和可能性评分。
- `src/reports.ts`：求职/招聘报告生成和 AI 分段解析。
- `src/plans.ts`：能力补强、简历执行、学生计划。
- `src/resumeImport.ts`：简历导入预览解析。
- `src/changelog.ts`：版本日志数据。
- `src/history.ts`：历史记录搜索与角色筛选。

## 6. 目前主要不足

### 6.1 `App.tsx` 仍偏大

虽然已经从 3700 多行降到约 2613 行，但仍然承担了不少页面组件和交互逻辑。后续应继续拆分：

- `AnalysisScreen`
- `ProfileScreen`
- `HistoryScreen`
- `PrivacyScreen`
- `ChangelogScreen`
- `QaWorkspace`

### 6.2 端到端测试仍不足

目前主要是单元测试和构建检查。后续应补真实用户流程测试：

- 游客进入。
- 选择求职/招聘。
- 填写问题。
- 清空示例。
- 生成报告。
- 打开 AI 配置。
- 保存历史记录。
- 导出报告。

### 6.3 报告导出格式偏基础

当前以文本导出为主，后续可增加：

- Markdown 导出。
- HTML 导出。
- PDF 导出。
- 适合发给用户或客户的正式报告模板。

### 6.4 UI 还需要系统验收

需要继续检查：

- 窄屏布局。
- 长文本卡片。
- 按钮文字溢出。
- 弹窗和轻提示遮挡。
- 历史记录数量较多时的体验。
- AI 错误状态和加载状态。

### 6.5 正式交付材料还不完整

建议后续补齐：

- 用户使用手册。
- AI 配置说明截图版。
- 测试反馈模板。
- 发布清单。
- 常见问题说明。

## 7. 下一步建议

优先级从高到低：

1. 继续拆分 `App.tsx` 页面组件，降低后续维护成本。
2. 增加端到端测试，覆盖真实用户主流程。
3. 完善正式报告导出格式，提升交付感。
4. 做一轮桌面 UI 走查，重点看长文本、窄屏和弹窗/提示。
5. 补充正式用户文档和测试反馈文档。
6. 若准备给更多人试用，重新打包 release 并输出发布清单。

## 8. 当前可交付状态

当前版本可以作为内测版交付给熟悉背景的测试用户使用。

建议交付方式：

- 给测试用户使用桌面安装包或 release 中的正式 exe。
- 同时附带使用说明。
- 明确说明这是内测版，重点收集使用流程、题目理解、报告价值和 AI 配置反馈。

暂不建议直接作为完全正式商业版大规模分发，原因是端到端测试、正式报告模板、用户文档和体验验收还需要继续补强。
